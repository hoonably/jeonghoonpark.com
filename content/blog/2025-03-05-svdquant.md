style="list-style-type:circle">Transformer 기반 모델에서 Self-Attention 연산이 많아서 <strong>활성화값(Activation)의 범위가 넓어지고 이상치가 발생</strong>하는 경우가 많음.</li></ul></div><div dir="auto" style="display:contents"><ul class="bulleted-list" id="331451cf-7b79-80cc-9b10-e3b8804e607e"><li style="list-style-type:circle">이를 8-bit이나 4-bit로 양자화하면, 작은 값들은 모두 0이 되고, 정보 손실이 심해짐.</li></ul></div></li></ul></div><div dir="auto" style="display:contents"><ul class="bulleted-list" id="331451cf-7b79-80b3-abc4-c25105111182"><li style="list-style-type:disc"><strong>해결 방법</strong><div dir="auto" style="display:contents"><ul class="bulleted-list" id="331451cf-7b79-80bd-b13a-f13683a12a78"><li style="list-style-type:circle">활성화값(Activation)의 채널별 스케일링을 적용하여, <strong>이상치를 가중치(Weight) 쪽으로 이동</strong>시킴.</li></ul></div><div dir="auto" style="display:contents"><ul class="bulleted-list" id="331451cf-7b79-805b-b167-d6430d7339ae"><li style="list-style-type:circle">즉, 원래 Activation 값이 크면, 해당 채널을 스케일링해서 줄이고, 대신 그 값을 Weight에서 보상해주는 방식.</li></ul></div><div dir="auto" style="display:contents"><ul class="bulleted-list" id="331451cf-7b79-80b9-854d-c976c23a18d2"><li style="list-style-type:circle">이렇게 하면, <strong>Activation 값이 양자화할 때 손실 없이 더 균등하게 분포할 수 있음</strong>.</li></ul></div></li></ul></div><div dir="auto" style="display:contents"><ul class="bulleted-list" id="331451cf-7b79-806b-886f-f6e34dd5ae40"><li style="list-style-type:disc"><strong>한계</strong><div dir="auto" style="display:contents"><ul class="bulleted-list" id="331451cf-7b79-80f5-91c4-c866fba5945b"><li style="list-style-type:circle">Weight 쪽으로 이상치를 몰아넣으면, <strong>Weight의 값이 커지고, Weight 양자화 시 오류가 커질 가능성</strong>이 있음.</li></ul></div><div dir="auto" style="display:contents"><ul class="bulleted-list" id="331451cf-7b79-8025-93b2-ca61845cebc7"><li style="list-style-type:circle">따라서 <strong>Weight를 4-bit로 양자화할 경우 정보 손실이 발생할 수 있음</strong>.</li></ul></div></li></ul></div><div dir="auto" style="display:contents"><hr id="331451cf-7b79-8079-8f25-c705110c3fae"/></div><div dir="auto" style="display:contents"><h3 class="" id="331451cf-7b79-8021-b958-eeca5e90c399"><strong>2. AWQ (Activation-aware Weight Quantization, 2024) – Weight에서 Activation으로 이상치 이동</strong></h3></div><div dir="auto" style="display:contents"><p class="" id="331451cf-7b79-80b2-9cc6-e06e91f365f5">AWQ는 <strong>Weight의 이상치를 줄이기 위해 Activation으로 분산시키는 방식</strong></p></div><div dir="auto" style="display:contents"><ul class="bulleted-list" id="331451cf-7b79-80a5-bfc0-fa7b7ff3bbc2"><li style="list-style-type:disc"><strong>기존 문제</strong><div dir="auto" style="display:contents"><ul class="bulleted-list" id="331451cf-7b79-8010-bb3d-cd20aa122169"><li style="list-style-type:circle">SmoothQuant 방식처럼 이상치를 Weight 쪽으로 이동시키면, Weight의 크기가 커져서 <strong>Weight를 4-bit로 양자화할 때 정보 손실이 발생</strong>할 가능성이 높아짐.</li></ul></div><div dir="auto" style="display:contents"><ul class="bulleted-list" id="331451cf-7b79-806b-88e1-d936b14ceccc"><li style="list-style-type:circle">특히, Weight에 이상치가 많으면, <strong>스케일링을 적용해도 양자화 오류가 커지고 성능이 떨어지는 문제</strong>가 발생.</li></ul></div></li></ul></div><div dir="auto" style="display:contents"><ul class="bulleted-list" id="331451cf-7b79-809f-94ec-ceb98173f1d7"><li style="list-style-type:disc"><strong>해결 방법</strong><div dir="auto" style="display:contents"><ul class="bulleted-list" id="331451cf-7b79-807d-8c6e-ffef1a656548"><li style="list-style-type:circle">대신 <strong>Weight에서 Activation으로 일부 이상치를 이동시켜서, Weight가 양자화될 때 정보 손실을 최소화</strong>함.</li></ul></div><div dir="auto" style="display:contents"><ul class="bulleted-list" id="331451cf-7b79-8033-ae6a-ddc3b606af8b"><li style="list-style-type:circle">즉, 중요한 Weight 값을 따로 보호하고, 불필요한 큰 값을 Activation 쪽으로 이동시켜서 Weight를 더 균등한 분포로 만들도록 설계.</li></ul></div></li></ul></div><div dir="auto" style="display:contents"><ul class="bulleted-list" id="331451cf-7b79-80a3-9945-f7ed0c737e87"><li style="list-style-type:disc"><strong>한계</strong><div dir="auto" style="display:contents"><ul class="bulleted-list" id="331451cf-7b79-8027-9f6b-e435a9cf5793"><li style="list-style-type:circle">Activation의 분포가 다시 넓어질 가능성이 있음 → <strong>Activation을 다시 4-bit로 양자화할 경우 문제가 발생할 수도 있음</strong>.</li></ul></div></li></ul></div><div dir="auto" style="display:contents"><hr id="331451cf-7b79-8084-be23-ee5d81a3a893"/></div><div dir="auto" style="display:contents"><h3 class="" id="331451cf-7b79-80c3-89f2-fc2777885bbc"><strong>3. SVDQuant (2024) – Outlier를 Low-Rank Component로 이동</strong></h3></div><div dir="auto" style="display:contents"><p class="" id="331451cf-7b79-8053-9da6-d7387f600201">SVDQuant는 SmoothQuant와 AWQ의 문제점을 모두 해결하려고, <strong>이상치를 이동시키는 것뿐만 아니라 Low-Rank Component로 흡수</strong>하는 방식</p></div><div dir="auto" style="display:contents"><ul class="bulleted-list" id="331451cf-7b79-802a-b0b6-e91b0235d220"><li style="list-style-type:disc"><strong>핵심 아이디어</strong><div dir="auto" style="display:contents"><ul class="bulleted-list" id="331451cf-7b79-80c4-bcd8-f629b3378630"><li style="list-style-type:circle">SmoothQuant처럼 <strong>Activation의 이상치를 Weight로 이동</strong>하면서도,</li></ul></div><div dir="auto" style="display:contents"><ul class="bulleted-list" id="331451cf-7b79-8008-943c-f37dcf0ce0b0"><li style="list-style-type:circle">AWQ처럼 <strong>Weight에서 다시 Activation으로 이동하는 대신, Low-Rank Component로 분리하여 저장</strong>.</li></ul></div><div dir="auto" style="display:contents"><ul class="bulleted-list" id="331451cf-7b79-8039-9692-e74a43ec1fd4"><li style="list-style-type:circle">즉, <strong>이상치를 양자화하지 않고, 16-bit Low-Rank Component로 유지</strong>하여 정보 손실을 최소화.</li></ul></div></li></ul></div><div dir="auto" style="display:contents"><ul class="bulleted-list" id="331451cf-7b79-800c-bdfa-da007aad1746"><li style="list-style-type:disc"><strong>장점</strong><div dir="auto" style="display:contents"><ul class="bulleted-list" id="331451cf-7b79-808d-9311-f84fb3aa1c61"><li style="list-style-type:circle">SmoothQuant나 AWQ처럼 <strong>한쪽으로 이상치를 몰아넣지 않고, Low-Rank Branch가 이상치를 흡수해서 손실을 막음</strong>.</li></ul></div><div dir="auto" style="display:contents"><ul class="bulleted-list" id="331451cf-7b79-8034-91f8-d53d9f0ec011"><li style="list-style-type:circle"><strong>Weight와 Activation 모두 균등한 분포를 가지게 되어, 양자화 오류가 줄어듦</strong>.</li></ul></div><div dir="auto" style="display:contents"><ul class="bulleted-list" id="331451cf-7b79-8058-a55e-f0395b976beb"><li style="list-style-type:circle">실제 실험에서도 <strong>SmoothQuant, AWQ보다 4-bit 양자화에서 성능이 뛰어남</strong>.</li></ul></div></li></ul></div><div dir="auto" style="display:contents"><hr id="331451cf-7b79-809a-8772-c5ba74bb3780"/></div><div dir="auto" style="display:contents"><h3 class="" id="331451cf-7b79-801a-83fd-e624f976ca9b"><strong>결론</strong></h3></div><div dir="auto" style="display:contents"><ul class="bulleted-list" id="331451cf-7b79-800e-83ac-e3e54a4567ac"><li <strong>Weight와 Activation에서 Low-Rank Component로 이동</strong> (이상치 자체를 제거하여 정보 손실을 최소화)</li></ul></div><div dir="auto" style="display:contents"><p class="" id="331451cf-7b79-80ff-8d5f-c5734710748b">즉,
